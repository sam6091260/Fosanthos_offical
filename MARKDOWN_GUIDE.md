# 文章 Markdown 語法指南

> 本指南說明在 `seed.js` 的 `content` 欄位中可使用的 Markdown 語法，以及在心光卉網站上的呈現效果。

---

## 基本規則

1. **content 文字必須頂格寫**（緊接 `` ` `` 後面），不能有前導空格
2. 空一行 = 換段落
3. 現有的純文字內容完全相容，不需要全部改寫

```js
// ✅ 正確寫法
content: `第一段文字。

第二段文字。`,

// ❌ 錯誤寫法（前面有空格 → 會變成程式碼區塊）
content: `
    第一段文字。
    
    第二段文字。`,
```

---

## 可用語法一覽

### 標題（適合課程、教學類文章）

```markdown
## 中標題（金色，最常用）
### 小標題
```

> ⚠️ 不建議用 `# 大標題`，因為文章已有 `<h1>` 標題。
> 學員分享類文章通常不需要標題，用分隔線即可。

---

### 粗體（強調重點）

```markdown
這是一段**重要的話**。
```

效果：這是一段**重要的話**。

---

### 斜體（輔助語氣）

```markdown
這是一段*柔和的語氣*。
```

效果：這是一段*柔和的語氣*。

---

### 引言區塊（關鍵對話、名言）

```markdown
> 「你的身體一直在等你回來。」
```

效果：左邊會出現金色直線，背景微微發亮。

**使用時機：**
- 人物對話的經典句
- 課程中的核心教誨
- 需要被記住的一句話

---

### 分隔線（段落情境轉換）

```markdown
---
```

效果：一條金色漸層線，分隔不同的敘事階段。

**使用時機：**
- 從「問題描述」→「轉折經歷」
- 從「過程」→「結果與感悟」
- 任何需要「喘息」的地方

---

### 無序列表

```markdown
- 呼吸練習
- 冥想引導
- 花波體驗
```

---

### 有序列表

```markdown
1. 祈禱的藝術
2. 傾聽的藝術
3. 看見的藝術
```

---

## 不同文章類型的 Markdown 策略

### 學員分享（student）

以**情感流動**為主，不用標題，用分隔線和引言：

```markdown
content: `開頭敘述問題和背景。

---

轉折經歷，遇見了改變。

> 「關鍵的一句話。」

描述改變的過程和**內心的感受**。

---

結尾感悟。

**最後一句有力的結語。**`
```

---

### 寶老師短文（teacher）

以**哲理層次**為主，用標題分段，引言點題：

```markdown
content: `開場提問或觀察。

## 第一個觀點

論述與故事。

> 核心金句。

## 第二個觀點

進一步的思考。

---

**結語，留給讀者思考的空間。**`
```

---

### 課程推廣（course）

以**資訊結構**為主，用標題和列表：

```markdown
content: `課程簡介。

## 🌸 課程特色

- 特色一
- 特色二
- 特色三

## 📋 課程內容

1. 第一天：主題
2. 第二天：主題
3. 第三天：主題

## ✨ 適合對象

- 想要自我探索的人
- 對身心靈有興趣的人

---

**名額有限，歡迎報名諮詢。**`
```

## 常見錯誤

| ❌ 錯誤 | ✅ 正確 | 原因 |
|---|---|---|
| `-🔮 標題` | `## 🔮 標題` | `-` 是列表，不是標題 |
| ` 🌍 標題`（前有空格） | `## 🌍 標題` | 前導空格會變程式碼 |
| `    文字開頭`（4格空白） | `文字開頭` | 4 空格 = 程式碼區塊 |
| `content: \`↵    文字` | `content: \`文字` | 反引號後直接接文字 |

---

## 快速參考卡

```
**粗體**          → 強調重點
*斜體*           → 柔和語氣
> 引言            → 金色引言框
---              → 金色分隔線
## 標題           → 金色中標題
### 小標題        → 深色小標題
- 項目            → 無序列表
1. 項目           → 有序列表
```
